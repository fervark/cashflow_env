#### Установка окружения ####

!! Для запуска команды на вашей локальной машине Linux\WSL должна быть доступна команда `make`

1. Отключите VPN так как он может мешать подключению к репозиториям
2. Создайте папку "cashflow" и склонируйте репозиторий cashflow_env
2. Перейдите в папку `cd cashflow_env`
3. Сосздайте .env файл из .env.example
4. Запустите команду `make init` для налала сборки окружения.
5. Перейти в директорию проекта `cd cashflow_app` и запустить команду миграций `goose up` 
6. Запустить приложение в директории `cmd/app/main.go`
7. Запустить Postmnan для проверки запросов

***
##### Подготовка инструментов #####
- выполнить миграции
- создать миграцию `cd cashflow_app` и `goose create <name> sql`
- выполнить миграцию `cd cashflow_app` и `goose up`

##### Структура приложения #####
- cmd (комаанды приложения)
- config (пакеты конфигурации)
- internal (ядро приложения)
- - app (пакеты приложения)
- - - endpoint (роуты приложения)
- - - factory (фабрики)
- - - getter (геттеры получения данных)
- - - model (модели данных)
- - - pkg (базовый пакет приложения app)
- - - service (сервисы ендпоинтов)
- - - setter (сеттеры для записи данных)
- - database (пакеты работы с базой данных)
- - middleware (базовые пакеты middleware)
- migrations (файлы миграций)

***
##### Endpoints #####
- MainPage (Method GET, Patch: '/'). Главная страница.
- SetCategory (Method POST, Patch: '/set/category'). Создание категории.
- GetTransactions (Method GET, Patch: '/transactions'). Получить список категории.
- SetTransaction (Method POST, Patch: '/set/transaction'). Создание транзакции.
- GetCashMovementStats (Method GET, Patch: '/movement-stats'). Получить статистику по транзакциям.

***
##### Параметры SetCategory  #####
- name (string)

##### Параметры GetTransactions  #####
- user_id (int). Обязательный. (В миграциях создан пользователь с ID = 1)
- type (string). Не обязательный, если не указан - выводятся все транзакции. Указывается тип доход `income` или расход `expense`
- page. Не обязательный. 
- limit. Не обязательный (по умолчанию выводится по 20 записей).
- date_from. Не обязательный, при указании даты от и до, происходит фильтрация по диапозону.
- date_to. Не обязательный.
- category_id. Не обязательный.

##### Параметры SetTransaction  #####
- user_id. Обязательный.
- category_id. Обязательный.
- type. Обязательный. (доход `income` или расход `expense`)
- price. Обязательный.

##### Параметры GetCashMovementStats  #####
- user_id. Обязательный.
- date_from. Не обязательный, при указании даты от и до, происходит фильтрация по диапозону.
- date_to. Не обязательный.
